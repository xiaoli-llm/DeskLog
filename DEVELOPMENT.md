### 3.2 优化过程

| 优化项 | 优化前 | 优化后 | 效果 |
|--------|--------|--------|------|
| 路由方案 | BrowserRouter | HashRouter | 兼容 Electron file:// 协议 |
| 数据库写入 | run() -> save() -> getLastInsertRowId() | this.db.run() -> getLastInsertRowId() -> save() | 修复 id=0 问题 |
| 状态管理 | currentLog 可能为 null | createEmptyLog 确保永不为 null | 消除空指针崩溃 |
| 请求竞态 | 无防并发机制 | fetchVersion 版本号 + selectedDate 校验 | 防止数据错乱 |
| 自动保存 | 使用闭包变量 todayTasks | 从 store.getState() 直接读取 | 始终保存最新数据 |
| 首次渲染 | auto-save useEffect 立即触发 | isInitialMountRef 跳过首次 | 避免无意义保存 |
| 分页体验 | 切换日期/项目分页不重置 | useEffect 监听切换重置 | 避免空白页 |
| 数据加载 | fetchTasks 和 fetchProjects 串行 | Promise.all 并行加载 | 加载速度提升约50% |
| 自动保存指示 | 竖向显示 | 水平 flex 布局 | 不影响 UI 布局 |
| 空状态 UI | 两段重复代码 | 单一三元表达式 | 消除重复按钮 |
